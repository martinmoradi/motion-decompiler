## Paradigms Worth Importing

- **Import id-decisive re-resolution, with a validator ladder.** For YoinkIt, the transferable pattern is not "selector string plus hope"; it is a durable snapshot plus a validated ladder: live ref, `{tag,id,classes,text}` snapshot, fresh DOM scan, and a forgiving fallback. This maps directly to `__cap.on(sel)` surviving HMR or re-rendered nodes.
- **Import expression recovery as provenance, not as a hard requirement.** The implementation gap around `{count} of {total}` makes the lesson sharper: YoinkIt should mark text as dynamic or source-bound when evidence supports it, but missed expression recovery should degrade to "possibly dynamic" rather than blocking capture or freezing misleading literals.
- **Import precise source hints as optional, verified provenance.** Astro hints are a good model for high-confidence source anchors in a spec, but only if verified against source text. Treat React `__source`, Vite plugin hints, or future Svelte hints as plug-ins, not assumptions.
- **Avoid source write-back mechanics.** The Svelte accept path is elegant but belongs to Impeccable's "agent owns repo and HMR" physics. YoinkIt should lift the contract and verification posture, not the route-file mutation.
- **Keep sidecar metadata in mind for framework pages.** The Svelte `params.json` workaround is a warning for capture metadata too: brace-bearing JSON in `data-*` attributes can break Svelte and confuse JSX-like compilers. Out-of-band stores are safer.
