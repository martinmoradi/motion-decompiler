## Deep Implementation Notes

- Confirmed: no jsdom dependency or canonical import. `package.json:80-90` lists runtime deps (`css-select`, `css-tree`, `domutils`, `fflate`, `htmlparser2`, `marked`) and optional `puppeteer`; no `jsdom`. A source search for jsdom imports under canonical `cli`, `extension`, `site`, `skill`, and `tests` found none. The remaining `jsdom` text is comments and historical naming, for example `checks.mjs:1703-1708`.
- Confirmed: `checkBorders` is a pure decision core. It only reads `tag`, `widths`, `colors`, and `radius`, skips `BORDER_SAFE_TAGS`, gates on width/chroma, side dominance, and radius, then emits `side-tab` or `border-accent-on-rounded` (`cli/engine/rules/checks.mjs:26-51`).
- Confirmed: browser and static adapters acquire measurements and call the same core. Browser uses `getBoundingClientRect` and `getComputedStyle` with a 20x20 layout bail (`checks.mjs:755-768`). Static reads computed-style-shaped properties and a pre-resolved radius (`checks.mjs:1718-1746`); the current static driver passes `overrides: null` and computes radius through `resolveBorderRadiusPx` (`engines/static-html/detect-html.mjs:91-93`).
- Confirmed: regex reimplements `side-tab`, so drift is possible there. `REGEX_MATCHERS` has six `side-tab` matchers for Tailwind side-border utilities, CSS left/right shorthands, width longhands, logical inline borders, logical inline widths, and JS style props (`engines/regex/detect-text.mjs:59-78`). Thresholds differ from the pure core in places, as the report says.
- Confirmed with recomputation: registry counts are still `44` total, `26` slop, `18` quality, `4` provider-gated, and `8` advisory. The four gated rules are `gpt-thin-border-wide-shadow`, `repeating-stripes-gradient`, `theater-slop-phrase`, and `image-hover-transform` (`registry/antipatterns.mjs:356-400`).
- Regex coverage recomputed from `REGEX_MATCHERS`, `TEXT_CONTENT_ANALYZER_IDS`, page analyzer ids, and design-system source scan is 20 rule ids: `ai-color-palette`, `aphoristic-cadence`, `border-accent-on-rounded`, `bounce-easing`, `broken-image`, `dark-glow`, `design-system-color`, `design-system-font`, `design-system-radius`, `em-dash-overuse`, `flat-type-hierarchy`, `gradient-text`, `gray-on-color`, `layout-transition`, `marketing-buzzword`, `monotonous-spacing`, `numbered-section-markers`, `overused-font`, `side-tab`, `single-font`. That matches the matrix's regex column.
- Confirmed: browser path runs broad and filters later. `collectBrowserFindings` explicitly says provider-gated rules are not filtered in browser because running checks is free (`browser/injected/index.mjs:1461-1464`). Node return paths filter with `filterByProviders` (`registry/antipatterns.mjs:430-438`), for example URL detection maps then filters at `engines/browser/detect-url.mjs:237-241`, static at `detect-html.mjs:226`, regex at `detect-text.mjs:552`.

## Paradigms Worth Importing

- Keep the trinity, but name the applicability boundaries precisely. For YoinkIt, that means pure analysis math plus runtime adapters, but also a matrix that says which runtime is authoritative, partial, or not applicable.
- Use a generated in-page bundle when YoinkIt grows beyond one file. Impeccable's build scripts prove the useful pattern: concatenate browser-safe modules, strip imports/exports, and write both the CLI/browser bundle and extension detector from canonical modules (`scripts/build-browser-detector.js:17-60`, `scripts/build-extension.js:27-82`).
- Steal provider-style emit filtering for capture confidence. Impeccable runs broad in the browser and filters/downranks at output; YoinkIt can capture all moved layers first, then decide which layers are high-confidence enough for the final spec.
- Steal the profiler shape as an observability contract: `{engine, phase, ruleId, target}` with p50/p95 summary maps cleanly to YoinkIt phases such as map, arm, trigger, sample, reduce, and emit.
- Upgrade the matrix idea with a partial symbol. The `cramped-padding` and `clipped-overflow-container` findings show why binary checkmarks hide important implementation truth: static can have useful non-layout coverage and still lack browser geometry.
