## Paradigms Worth Importing

- The most transferable pattern is not "carbonize CSS into a real stylesheet"; it is the tiny completion classifier plus durable unfinished phase. For YoinkIt, model this as `draft_ready` followed by `crystallize_required`, where the fast path persists the sampled frames quickly and the slow path normalizes layers/easings/selectors before a final `complete`.
- The report's `wrap` lesson should be framed as "bundle deterministic multi-step browser/agent work behind one command." For YoinkIt, that maps to a real-browser capture command that does settle, arm, trigger, wait, and dump in one deterministic operation while still requiring a visible tab.
- Preserve the event payload idea, but strengthen it for YoinkIt. Impeccable's current text fallback accepts first-match for short/dynamic text; YoinkIt probably wants stricter ambiguity reporting because capturing the wrong animated element corrupts the spec.
- The buffer-aware wrap seam is a useful design smell to import: when two loops can touch the same surface, the fast helper must understand pending user-visible edits or refuse with a structured error rather than operate on stale source.
